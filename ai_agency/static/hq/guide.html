Как давать задачи команде

Способ 1 — через дашборд (рекомендуется)

Кнопка «Задача команде» на дашборде → текст задачи → режим:

  • Lite (1–2 агента) — быстро, дёшево, простые задачи
  • Standard (3–5 агентов) — оптимально для большинства задач
  • Full (вся команда) — сложные задачи, полный анализ

Приблизительная стоимость за задачу: Lite ~$0.02–0.05 · Standard ~$0.05–0.15 · Full ~$0.15–0.40

Яндекс.Метрика

Шаг 1 — номер счётчика

  1. Открой metrika.yandex.ru
  2. Выбери счётчик для ai-delivery.shop
  3. Настройки (шестерёнка) → вкладка «Счётчик»
  4. Скопируй номер (8 цифр, например: 98765432)

Шаг 2 — OAuth-токен

  1. Открой авторизацию Яндекса (ссылка ниже)
  2. Разреши доступ
  3. Скопируй токен из URL (после #access_token=)

oauth.yandex.ru/authorize?response_type=token&client_id=1d0b9dd4d652455a9eb710d450ff456a

Шаг 3 — добавь в .env на сервере

Подключись через PuTTY к 89.22.235.144

nano /var/www/ai_agency/ai_agency/.env

Добавь строки:

YANDEX_METRIKA_TOKEN=твой_токен_здесь
YANDEX_METRIKA_COUNTER=твой_номер_счётчика

Сохрани в nano: Ctrl+O → Enter → Ctrl+X

Шаг 4 — перезапуск сервиса

systemctl restart ai-agency

Шаг 5 — проверка

Открой /hq/analytics.html → блок «Посетители» должен показать реальные данные.

Google Search Console

Шаг 1 — credentials

  1. console.cloud.google.com — проект
  2. APIs & Services → Enable APIs → Google Search Console API
  3. Credentials → Create Credentials → OAuth 2.0 Client ID
  4. Скачай JSON

Шаг 2 — загрузка на сервер

Через WinSCP положи файл в:

/var/www/ai_agency/ai_agency/data/google_credentials.json

Шаг 3 — .env

GOOGLE_CREDENTIALS_PATH=/var/www/ai_agency/ai_agency/data/google_credentials.json
GOOGLE_SEARCH_CONSOLE_SITE=https://ai-delivery.shop/

Яндекс.Вебмастер

Тот же OAuth-токен, что и для Метрики. В .env:

YANDEX_WEBMASTER_HOST_ID=https:ai-delivery.shop:443

Формат: протокол + домен + порт без слешей.

Аккаунт-менеджер

Что делает автоматически

  • Каждый день кроме заданного в .env (например четверга) в заданное время — список задач на день в Telegram
  • Напоминания о клиентах со скорым дедлайном
  • Непроверенные задания учеников
  • Просроченные оплаты

Расписание в .env

ACCOUNT_MANAGER_TIME=09:00
ACCOUNT_MANAGER_SKIP_DAYS=3  # 3 = четверг (0=пн, 1=вт, 2=ср, 3=чт, 4=пт)

CRM — как вести клиентов

Статусы клиентов

  • lead — потенциальный клиент, ещё не платит
  • active — активный проект
  • paused — на паузе
  • done — проект завершён
  • cancelled — отменён

Этапы проекта (последовательно)

  1. Discovery — бриф, понимание задачи, scope
  2. Design — дизайн решения, архитектура
  3. Development — разработка
  4. Testing — тестирование с клиентом
  5. Launch — запуск и передача
  6. Support — поддержка после запуска

Критерии «этап завершён»

Discovery → есть подписанный бриф / scope

Design → клиент согласовал решение

Development → всё работает технически

Testing → клиент подтвердил, что всё ок

Launch → передан доступ, написана инструкция

Support → прошло 30 дней без критических проблем

Задачник и связка с AI-командой

Теперь быстрые задачи с дашборда и основной задачник работают на одной модели tasks_v2.

Как создавать задачи

  • На дашборде есть блок «Быстрая задача» для мгновенного добавления
  • Полный список, фильтры и статусы находятся в задачнике
  • Каждая задача может содержать описание, приоритет, дату выполнения и дедлайн

Две разные даты

  • Дата выполнения — когда задача должна быть сделана по плану
  • Дедлайн — крайний срок, после которого задача считается просроченной
  • В списках и на дашборде система сначала ориентируется на дату выполнения, а контроль просрочки идёт по дедлайну

Отправка в AI-команду

  • Кнопка «В AI-команду» создаёт связанную AI-задачу без потери контекста
  • После ответа у задачи появляется превью результата и ссылка «Открыть ответ»
  • Из задачника и из истории можно перейти сразу в нужный чат агента

История и контроль

  • В AI Команде есть история связанных задач
  • Там видно: запрос, исполнитель, статус, дату и краткий ответ
  • Если ответ ещё не готов, ссылка всё равно ведёт в правильный диалог агента
