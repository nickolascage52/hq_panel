- HI-3: sprints empty + sprints with data
- HI-6: tokens_used column present + in run response

Total pytest: **14 tests, all passing in ~94s**.

### Rate Limit UI Indicator

`GET /api/pipeline/rate-limits` endpoint:
- Per-model state (opus/sonnet/haiku) с pct, reset_at, last_updated
- Derived `status`: ok / warning (>=70%) / critical (>=90%)
- `should_pause` flag

`pipeline.html` topbar теперь показывает реальные данные:
```
🟢 opus 0% · 🟢 sonnet 0% · 🟢 haiku 0%
```
или при перегрузке:
```
⛔ ALL LIMITS REACHED   🔴 opus 95% · 🔴 sonnet 92% · 🔴 haiku 91%
```

Auto-refresh каждые 60 сек (rate-limit slow-changing).

### Telegram Pipeline Commands

`pipeline/telegram_commands.py` — extension module (НЕ переписывает telegram_bot.py, только добавляет handlers).

6 новых команд для владельца:
- `/pipelines` — список последних 10 runs с status и phase
- `/run <id>` — детали + последние 5 событий
- `/approve <id>` — approve awaiting_approval
- `/pause <id>` — pause running
- `/resume <id>` — resume paused/awaiting
- `/abort <id>` — abort run

Все защищены TELEGRAM_OWNER_ID — другие user'ы получают «⛔ Только для владельца».

В `telegram_bot.py` 1 try/except блок добавлен после existing handlers, который вызывает `register_pipeline_commands(app_builder)`. **Existing команды не тронуты.**

`OWNER_GUIDE.md` обновлён — таблица AI Pipeline команд.
