## Архитектурные изменения в v1.1

### Backend (новое в `pipeline_api.py`)

```
GET /api/pipeline/runs/{id}/sprints       — pipeline_sprints rows + spec_md
GET /api/pipeline/runs/{id}/files         — list of .md files в workspace
GET /api/pipeline/runs/{id}/files/{path}  — content of file (with traversal protection)
POST /api/pipeline/runs/{id}/pause        — manual pause (status='paused_user')
POST /api/pipeline/runs/{id}/resume       — resume from any paused state
POST /api/pipeline/runs/{id}/abort        — terminate (status='aborted')
```

### Backend (DB)

- Migration `_migrate_pipeline_v1_1()` — добавляет `pipeline_runs.tokens_used INTEGER DEFAULT 0`
- ClaudeRunner после каждого call инкрементирует cumulative tokens_used per run

### Frontend (new + updated)

- `hq-pipeline.js`: `listSprints`, `listFiles`, `getFileContent`, real `pauseRun/resumeRun/abortRun`
- `connectWebSocket` — переписан с exp backoff + missed-events replay
- `pipeline.html`: tokens badge в карточках (🪙 N)
- `pipeline-run-detail.html`:
  - tokens badge в header
  - Documents tab — split-view (список файлов слева, рендер справа) с marked.js
  - Sprints tab — list cards с progress bar + collapsible spec_md
  - Pause/Resume/Abort активные кнопки с confirm() для destructive

### Standards (new)

- `agency/standards/telegram_bot.md` — aiogram 3.x stack, FSM, systemd template
- `agency/standards/n8n.md` — self-hosted, workflow JSON delivery, common patterns
- `agency/standards/ai_assistant.md` — FastAPI + Claude + Qdrant, streaming responses
